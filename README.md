# Проєкт "Кулінарна книга" — MVVM версія (lab8)

Цей проєкт є розвитком попередньої версії застосунку (lab7), створеного на базі **WinUI 3** та **Entity Framework Core** (SQLite). Головна мета цієї версії — переведення проєкту на професійний архітектурний патерн **MVVM (Model-View-ViewModel)**.

## Що змінилося порівняно з минулим проєктом (lab7)

1. **Впровадження патерну MVVM:** Логіка програми була повністю відділена від візуального інтерфейсу. Code-Behind файл вікна (`MainWindow.xaml.cs`) тепер практично порожній і не керує елементами безпосередньо.
2. **Заміна подій на Команди (Commands):** Замість обробників подій на кшталт `Click="AddButton_Click"`, тепер використовуються команди `Command="{Binding AddCommand}"`.
3. **Data Binding (Прив'язка даних):** Дані передаються між інтерфейсом та логікою автоматично за допомогою механізму прив'язки (`{Binding ...}`).
4. **Використання `ObservableCollection`:** Замість звичайного `List`, для списку рецептів використовується `ObservableCollection`. Це дозволяє інтерфейсу автоматично оновлюватися при додаванні або видаленні елементів.
5. **Новий функціонал:** Додано автоматичний підрахунок загальної кількості рецептів та можливість сортування списку за алфавітом.

## Логіка роботи проєкту

Застосунок працює за схемою MVVM:
* **Model (Модель):** Відповідає за структуру даних та взаємодію з базою даних SQLite (EF Core).
* **View (Представлення):** XAML-розмітка, яка лише відображає дані та передає дії користувача (натискання кнопок, введення тексту) до ViewModel за допомогою Binding. Жодного програмного налаштування UI-компонентів у C#-коді немає — всі елементи знаходяться на формі.
* **ViewModel (Модель представлення):** Проміжний шар, який містить бізнес-логіку. Він отримує дані з Model, готує їх для View, обробляє команди від користувача та сповіщає View про зміни даних через інтерфейс `INotifyPropertyChanged`.

## Структура файлів та їх призначення

### 1. `CookbookData.cs` (Шар Model)
Містить структуру бази даних:
* Класи-сутності **`Category`** та **`Recipe`**.
* **`CookbookDbContext`** — контекст бази даних Entity Framework Core для підключення до локального файлу `cookbook_mvvm.db`.

### 2. `MvvmHelpers.cs` (Допоміжні класи MVVM)
Містить базові класи для забезпечення роботи патерну:
* **`ViewModelBase`**: Реалізує інтерфейс `INotifyPropertyChanged`. Використовується для сповіщення інтерфейсу про те, що значення якоїсь змінної оновилося і XAML має перемалювати дані.
* **`RelayCommand`**: Реалізує інтерфейс `ICommand`. Перетворює дії користувача (наприклад, клік по кнопці) у виклик відповідного методу у ViewModel.

### 3. `MainViewModel.cs` (Шар ViewModel)
Головний клас логіки програми. 
* Містить властивості для прив'язки (наприклад, `NewRecipeTitle` для вводу тексту, `TotalCount` для кількості).
* Зберігає список рецептів у `ObservableCollection<Recipe> Recipes`.
* Містить логіку ініціалізації бази даних, завантаження даних (`LoadData()`), додавання нових рецептів (`AddRecipe()`) та їх сортування (`SortRecipes()`).

### 4. `MainWindow.xaml` (Шар View)
Файл розмітки інтерфейсу:
* Містить елементи керування (`ListView`, `TextBox`, `Button`).
* Всі значення атрибутів `Text` або `ItemsSource`, а також дії кнопок (`Command`), жорстко прив'язані до відповідних властивостей у `MainViewModel` за допомогою синтаксису `{Binding ...}`.

### 5. `MainWindow.xaml.cs` (Code-Behind)
Максимально спрощений файл. Його єдина задача — встановити джерело даних (`DataContext`) для вікна при його ініціалізації: `this.DataContext = new MainViewModel();`. Більше жодної логіки тут немає.
